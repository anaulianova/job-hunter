# Mode: /track

**Purpose:** Sync job data between your local pipeline and the Google Sheets tracker. Keeps your spreadsheet up to date after evaluations, applications, and status changes.
**Input:** `/track` — runs the appropriate sync based on context, or ask for a specific action.

---

## What the tracker does

Your pipeline lives in two places: `pipeline/pipeline.json` (local, on your computer) and your Google Sheet (shareable, visible, the thing you check daily). `/track` moves data from the local pipeline to the sheet. It never overwrites anything you've edited in the sheet manually — it only adds new rows.

The Google Sheet has 4 tabs:

| Tab | What it contains |
|---|---|
| **Tracker** | Roles you've applied to — with status (Sent → Interview → Offer) |
| **Pipeline** | Every role you've evaluated, including skips and Tier 3s |
| **Job Postings** | URLs you want evaluated (paste here instead of the chat) |
| **App Q&A** | Application answers generated by `/apply` — one row per application |

### Sheet columns

**Tracker** (7 cols): Company · Job Title · Status · Application Date · Medium · URL · Location

**Pipeline** (8 cols): Company · Job Title · Tier · Match Score · Evaluated Date · Location · URL · Notes

**Job Postings** (5 cols): URL · Company · Job Title · Date Added · Status

**App Q&A** (9 cols): Company · Job Title · Tier · Date · Salary (free text) · Salary (number) · Why this company · Note to hiring manager · Anything additional

## Commands

### First-time setup (run once)

Creates the 4 tabs, adds column headers, colour coding, and dropdowns. Only needed when setting up a new Google Sheet.

```
uv run scripts/sheets.py --setup
```

**Plain English:** "Build my Google Sheet from scratch."

Formatting applied automatically:
- Header row bold and frozen on all tabs
- **Tracker** status dropdown with colour-coded cells: `Sent` (pink) · `First Screening` (yellow) · `Interview` (blue) · `Case Study` (orange) · `Rejected` (red, row greyed) · `Silent Rejection` (dark grey) · `Offer` (green)
- **Pipeline** rows colour-coded by tier: Tier 1 (green) · Tier 2 (yellow) · Tier 3 (orange) · Skip (red)
- **Job Postings** status dropdown: `Pending` (yellow) · `Evaluating` (orange) · `Skipped` (red) · `Evaluated` (green)

> **Run once only.** Re-running stacks conditional formatting rules. To reset: delete all 4 tabs manually in Google Sheets, then re-run `--setup`.

---

### After evaluating a role

Pushes all evaluated roles from `pipeline.json` to the Pipeline tab (Sheet 2). Run this after any `/evaluate` session.

```
uv run scripts/sheets.py --sync-pipeline
```

**Plain English:** "Update my pipeline tab with roles I've just evaluated."

Appends all roles except those in `discovered` status. Deduplicates by URL — running it twice is safe.

---

### After submitting an application

Pushes applied roles to the Tracker tab (Sheet 1). Called automatically by `/apply` — but run it manually if you applied somewhere outside the system.

```
uv run scripts/sheets.py --sync-tracker
```

**Plain English:** "Add this application to my tracker."

Appends roles with statuses: `applied` · `screening` · `interview` · `offer` · `rejected` · `withdrawn` · `cv_tailored`. Deduplicates by URL.

---

### Update a role's status

When something changes — you hear back, get rejected, have a call — update the status in the Tracker tab. Replace the URL and status with the actual values.

```
uv run scripts/sheets.py --update-status "https://company.com/jobs/123" "First Screening"
```

Valid statuses: `Sent` · `First Screening` · `Interview` · `Case Study` · `Rejected` · `Silent Rejection` · `Offer`

**Plain English:** "I got a response — update the status for this role."

---

### Evaluate URLs from the Job Postings tab

If you've pasted URLs into Sheet 3 (Job Postings), this prints the ones waiting to be evaluated.

```
uv run scripts/sheets.py --get-pending-urls
```

Then evaluate each URL with `/evaluate {url}`, and mark it done:

```
uv run scripts/sheets.py --mark-evaluated "https://company.com/jobs/123"
```

**Plain English:** "Show me what's in my to-evaluate queue." / "Mark this one as done."

---

### Sync application Q&A answers

Pushes the answers generated by `/apply` to the App Q&A tab (Sheet 4). Called automatically by `/apply` — you don't usually need to run this manually.

```
uv run scripts/sheets.py --sync-qa "Company Name" "Job Title" '{"salary_narrative": "...", "why_company": "..."}'
```

---

### Fix formatting issues

If bold formatting bleeds from headers into data rows (a Google Sheets quirk after append):

```
uv run scripts/sheets.py --fix-bold
```

If the Pipeline tab is missing tier colour coding (green/yellow/orange/red rows):

```
uv run scripts/sheets.py --fix-pipeline-formatting
```

## When /track is called by other modes

| Trigger | What runs automatically |
|---|---|
| After `/evaluate` completes | `--sync-pipeline` |
| After `/apply` confirms | `--sync-tracker` + `--sync-qa` |
| Manual status update request | `--update-status` |

## Requirements

- `config/google_creds.json` — Google service account credentials (gitignored, never committed)
- `GOOGLE_SHEET_ID` in `.env` — the ID from your Google Sheet URL: `docs.google.com/spreadsheets/d/{THIS_PART}/edit`

---

## Troubleshooting

**"Google credentials not found"** — make sure `config/google_creds.json` exists. Download it from Google Cloud Console → Service Accounts → your service account → Keys. See the full setup instructions at the top of `scripts/sheets.py`.

**"GOOGLE_SHEET_ID not set"** — add `GOOGLE_SHEET_ID=your_id_here` to your `.env` file.

**"URL not found in Tracker"** — the URL in your pipeline.json must match exactly what's in the sheet. Copy the URL from the pipeline file if unsure.

**Missing dependencies** — run `uv pip install google-auth google-auth-httplib2 google-api-python-client python-dotenv` if you see an import error.
